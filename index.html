Sumário

Introdução

  • JavaScript é uma linguagem de programação Web via Scripts
  • Quase todos os sites modernos a utilizam
  • Possui suporte aos browsers por meio de um interpretador
  • Junto com HTML e CSS, JS forma a tríade do desenvolvimento Web

Contexto

  • Os browsers eram estáticos
  • A Netscape surgiu com o intuito de deixar a web mais dinâmica
  • Implementação: Scheme x Java

Criador

  • Brendan Eich, americano criador do JavaScript
  • Em maio de 1995 criou um protótipo de uma linguagem, chamada de Mocha
  • Essa linguagem foi renomeada para LiveScript
  • Em dezembro de 1995, a linguagem foi renomeada para JavaScript

Normalização

  • Em agosto de 1996 a Microsoft criou uma linguagem idêntica ao JS
  • A Netscape decidiu normalizar a linguagem através da organização ECMA Internacional
  • Em novembro de 1996 começaram os trabalhos para a criação da normativa ECMA-262
  • Contudo, o nome JavaScript já estava patenteado pela Sun Microsystems
  • Dessa forma, o verdadeiro nome do JS é ECMAScript
  • Essa linguagem é mantida pelo comitê TC39, composto por especialistas de grandes empresas

Paradigma

  • É um "estilo" de programação

  • Multi-paradigma

  • Imperativo

    - Procedural

    - Orientado a Objetos

  • Lógico

    - Funcional

Paradigma - Imperativo

  • Variáveis

    - Forma pelo qual modelamos as células da memória

  • Comandos de atribuição

    - Operações de transferência dos dados e instruções

  • Forma iterativa de repetição

Procedural x Orientado a Objetos

Paradigma - Funcional

  • Função

    - Forma pelo qual relacionamos dois conjuntos com uma regra para associar ambos

  • Evitar Efeitos colaterais

    - Mudança no estado da aplicação que ocorreu fora do escopo pelo qual sua função foi chamada.

  • Imutabilidade

    - Evitar valores de variáveis, seja por atribuições ou por funções

JavaScript x Haskell

Características

  • Case Sensitive

    - Diferencia maiúsculas e minúsculas

  • Tipagem dinâmica

    - Não precisamos definir o seu tipo

  • Tipagem fraca

    - Os tipos são convertidos automaticamente conforme a necessidade

  • Interpretado

    - O código é executado na ordem de cima para baixo

    - O resultado da execução é imediatamente retornado

Características

  • Avaliação “Lazy” (Preguiçosa)

    - A validação de uma parte do código só ocorre se ela for necessária para ser executada

  • Orientação a Objetos

    - Conceitos como polimorfismo e herança podem ser implementados de maneira bem eficiente e simples

  • Tipos Básicos

    - Boolean, null, undefined, number, string e symbol;

  • Tipos Compostos

    - Objetos, arrays e funções

Características

  • Escopo

    - Escopo local

    - Escopo global

  • Encapsulamento

    - Podemos encapsular um código e exportar entidades.

    - Chamamos este encapsulamento de módulo
